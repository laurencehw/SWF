# Contributing to the US Sovereign Wealth Fund White Paper

Thank you for your interest in contributing to this white paper. This document provides guidelines for contributing to improve the analysis, correct errors, or extend the research.

## Types of Contributions Welcome

### 1. Factual Corrections
- Errors in data, statistics, or citations
- Outdated information that needs updating
- Mischaracterizations of existing funds or policies

### 2. Content Improvements
- Additional relevant research or citations
- Comparative analysis from other countries
- Updated market data or fund performance information
- Legal or regulatory analysis

### 3. Clarity and Readability
- Typographical errors
- Unclear passages that need rewriting
- Formatting improvements

### 4. Translations
- Executive summary translations for international audiences

### 5. Extended Analysis
- Deeper dives into specific sections
- Case studies of individual sovereign wealth funds
- Modeling or quantitative analysis

## How to Contribute

### For Minor Corrections (typos, small errors)

1. Open an issue describing the correction needed
2. Reference the specific section and line if possible
3. Provide the correct information with sources if applicable

### For Substantive Contributions

1. **Fork the repository**
2. **Create a feature branch** (`git checkout -b improvement/your-topic`)
3. **Make your changes** to the Markdown source file
4. **Document your sources** â€” all factual claims should be cited
5. **Submit a pull request** with a clear description of changes

### For New Analysis or Extensions

1. Open an issue first to discuss the proposed addition
2. Describe the scope and value of the new content
3. Wait for feedback before investing significant effort

## Style Guidelines

### Writing Style
- Maintain academic/policy paper tone
- Use active voice where possible
- Define acronyms on first use
- Avoid partisan political language
- Focus on analysis, not advocacy

### Citations
- Use footnote style consistent with existing paper
- Prefer primary sources (government documents, fund reports)
- Academic papers should be peer-reviewed where possible
- News sources acceptable for recent events

### Data Standards
- All statistics should include sources
- Clearly indicate data vintage (year of data)
- Distinguish between estimates and official figures
- Use consistent units (billions USD)

### Formatting
- Follow existing Markdown conventions
- Tables should use the existing style
- Section numbering follows Roman numerals for main sections
- Preserve pandoc/LaTeX compatibility

## Review Process

1. All contributions will be reviewed for accuracy and relevance
2. Factual claims will be verified against cited sources
3. Feedback will be provided on pull requests
4. Significant changes may require discussion before merging

## Code of Conduct

### Be Respectful
- This is a policy analysis document, not a political platform
- Critique ideas, not people or institutions
- Maintain professional discourse

### Be Accurate
- Double-check facts and figures
- Acknowledge uncertainty where it exists
- Don't overstate conclusions

### Be Constructive
- Offer solutions when identifying problems
- Explain reasoning behind suggestions
- Help improve the overall quality of analysis

## Questions?

If you have questions about contributing, please open an issue with the label "question" and we'll respond as soon as possible.

## Recognition

Contributors who provide substantive improvements will be acknowledged in the paper's acknowledgments section (with permission).
